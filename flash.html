OSI Model Question 1: What conceptual framework is used to understand and describe how different networking protocols and technologies interact within a network?
Answer: OSI (Open Systems Interconnection) model.
OSI Model Question 2: How many layers are there in the OSI model, each responsible for specific tasks?
Answer: Seven layers.
OSI Model Question 3: What does the Physical Layer of the OSI model deal with?
Answer: Physical connections, cables, and hardware.
OSI Model Question 4: Which layer of the OSI model manages data frames, error detection, and access control, dealing with Ethernet and MAC addresses?
Answer: Data Link Layer.
OSI Model Question 5: What does the Network Layer handle in the OSI model?
Answer: Routing, addressing, and logical addressing (IP).
OSI Model Question 6: Which OSI layer ensures end-to-end communication, error checking, and flow control, using protocols like TCP and UDP?
Answer: Transport Layer.
OSI Model Question 7: What is the main responsibility of the Presentation Layer in the OSI model?
Answer: Translates data into a common format, which includes encryption and compression.
OSI Model Question 8: In the OSI model, which layer supports user applications like HTTP and FTP?
Answer: Application Layer.
DHCP/TCP-IP Question 1: What does DHCP stand for, and what is its primary function in a network?
Answer: DHCP stands for Dynamic Host Configuration Protocol, and it's used to automatically assign IP addresses and network configuration settings to devices.
DHCP/TCP-IP Question 2: What does TCP/IP stand for, and what are the two main protocols in this suite?
Answer: TCP/IP stands for Transmission Control Protocol/Internet Protocol. The two main protocols are TCP (Transmission Control Protocol) and IP (Internet Protocol).
DHCP/TCP-IP Question 3: How does DHCP enhance network configuration within the TCP/IP suite?
Answer: DHCP assigns IP addresses, subnet masks, default gateways, and DNS server addresses to devices, making network configuration easier and more efficient.
Switches, Bridges, and Nodes Question 1: What is the primary difference between switches and bridges in networking, and which OSI layer do they operate at?
Answer: Switches are more advanced and efficient than bridges and operate at the data link layer (Layer 2) of the OSI model.
Switches, Bridges, and Nodes Question 2: How do switches and bridges determine where to forward Ethernet frames in a network?
Answer: They examine MAC addresses in Ethernet frames and forward frames only to the appropriate port where the destination device is located.
Switches, Bridges, and Nodes Question 3: What is the primary advantage of switches over bridges in reducing network congestion?
Answer: Switches can make forwarding decisions more intelligently based on MAC address tables.
Switches, Bridges, and Nodes Question 4: What does the term "node" refer to in networking, and how are nodes assigned MAC addresses?
Answer: A "node" refers to any device connected to a network, and they are assigned MAC addresses to communicate within a LAN.
